# 項目規劃

## 可行性驗證版（Feasibility / v0）

> 目標：用最小成本驗證核心閉環——**建立動作 → (可選)排課 → 訓練時逐組記錄 → 檢視歷史**。
> 平台：**iOS 原生（SwiftUI + SwiftData）**，local-first、單機即可用。
> 架構：**Clean Architecture + 多模組（每個 domain 一個 SPM package，編譯期切開）**，詳見 [`ARCHITECTURE.md`](./ARCHITECTURE.md)。

### 已定決策
- **平台**：iOS 原生（SwiftUI + SwiftData）；不走 Web
- **架構**：Clean Architecture，每個 domain 一個 SPM package，Domain 層純 Swift、可抽換可測
- **後端**：獨立 repo、Go 撰寫；以 OpenAPI 產生 Swift client 給 iOS 用，兩端徹底切開。**v0 不做後端**（純本地單機）
- **同步 / 分享**：本地優先——v0 純本地；日後接 Go 後端才有「登入跨裝置同步」與「選擇公開 spec/菜單」，屆時一個訓練場次完成時整包上傳
- **紀錄可脫離課表**：課表僅為預填來源，臨時加練 / 不排課也能直接記錄
- **組(set)粒度**：逐組記錄（每組各存重量與次數）
- **肌群分類**：enum 固定清單 `胸 / 背 / 腿 / 肩 / 手臂 / 核心 / 功能性訓練 / 其他`
- **重量單位**：可自由切換；每筆重量存「數值 + 輸入當下的單位」為真實來源，切換只影響顯示、不改動已存資料
- **開源與發行**：開源（Apache-2.0）；先用免費簽章 build 自用 → 功能齊再走 TestFlight / 上架（屆時才需 Apple Developer $99/年）
- **範圍**：單用戶、先不做登入/多用戶

### 範圍（做 / 先砍）
- ✅ 做：動作 CRUD、排課(可選)、記錄場次(逐組·本地暫存·完成批次上傳)、按動作/按日期查歷史
- ✂️ 先砍：多用戶/登入、範本複製、進度圖表、資料匯出、休息計時、缺課提醒、智能推薦

### 建議建置順序
1. **Spec 動作 CRUD** — 動作庫是後續一切的基礎
2. **記錄訓練場次** — 逐組記錄 → 本地暫存 → 完成批次上傳（核心價值先驗證）
3. **歷史檢視** — 按動作看歷次紀錄與進展
4. **排課(預填)** — 最後補；因紀錄已可獨立運作

## MVP（v1.0）- 最小可行版本

### 第一階段：基礎管理
**訓練項目庫（Spec）**
- [ ] 新增訓練動作
- [ ] 瀏覽所有動作清單
- [ ] 編輯動作資訊
- [ ] 刪除動作

**個人課表（Personal）**
- [ ] 為指定日期建立訓練課表
- [ ] 從項目庫選擇動作並設定參數（重量、次數、組數、休息時間）
- [ ] 瀏覽特定日期的課表
- [ ] 編輯已排課表

### 第二階段：訓練執行紀錄
**訓練紀錄（TrainingRecord）**
- [ ] 查看當日課表
- [ ] 逐項記錄實際完成情況（次數、重量）
- [ ] 記錄訓練狀態（完成/跳過/中斷）
- [ ] 記錄訓練感受評分
- [ ] 記錄訓練備註

### 第三階段：基礎查詢
**訓練紀錄檢視**
- [ ] 查看單個動作的歷史紀錄
- [ ] 按日期範圍查詢訓練紀錄
- [ ] 簡單統計（某動作做過多少次）

## 用戶使用流程簡述

### 初期設置
1. 建立訓練項目庫（新增常做的動作：臥推、深蹲、硬舉等）

### 課表安排（每週/每月一次）
1. 選擇日期
2. 為該日添加訓練項目
3. 設定目標參數（重量、次數、組數、休息）
4. 保存課表

### 訓練執行（每訓練日）
1. 打開當日課表
2. 逐項完成訓練並記錄實際數據
3. 標記完成狀態和感受

### 進度檢視（週末或月末）
1. 查看某動作的歷史紀錄
2. 檢視本週訓練完成情況
3. 調整下週課表
